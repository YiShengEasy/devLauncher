# 需求说明

## 背景与问题

开发、测试和运维任务经常写在 README 或内部操作文档中。任务有说明但没有统一入口，使用者需要先打开项目、搜索 Markdown、复制命令，再切换到终端；同一任务也难以复用到 DevLauncher 工作流和键位。

Runme 已经提供了“Markdown 代码块即任务”的外部执行模型。本功能只负责发现、呈现和连接，不重新实现 Runme 的执行内核。

## 用户与场景

| 用户 | 场景 | 期望结果 |
| --- | --- | --- |
| 开发者 | 从项目 README 找到启动、构建、测试命令 | 扫描后看到可执行任务，直接打开终端 |
| 测试人员 | 复用项目中的 smoke、lint、contract 任务 | 按分类筛选并预览命令，减少复制错误 |
| 运维人员 | 复核发布、迁移、清理任务 | 看到风险提示，执行前明确确认 |
| 团队负责人 | 将标准任务绑定到 DevLauncher 键位 | 一键把任务保存为普通工作流 |

## 功能需求

### P0

- 选择一个本地项目目录。
- 扫描目录下的 Markdown 文件，跳过 .git、依赖、构建产物等目录。
- 调用 Runme 的 `list --json` 发现显式命名的 Markdown 任务，并显示任务名、文件、行号、语言和命令内容。
- 显示 Runme CLI 是否可用和版本信息。
- 执行前校验任务文件仍位于项目目录内，避免路径越界。
- 通过现有终端窗口执行 runme run，保留终端输出和交互能力。
- 将选中的任务保存为单步工作流。

### P1

- 按环境、开发、测试、构建、发布、部署、数据、运维分类；支持带数量的类型筛选，并在“全部”视图中按类型分组。
- 根据命令关键词展示安全、需复核、高风险状态。
- 支持复制经校验的完整命令。
- 记录最近扫描过的项目，最左侧以稳定顺序展示项目名、路径和任务数；点击项目优先使用当前窗口会话缓存瞬时切换，刷新按钮才重新扫描，支持只移除本地历史索引。
- 扫描为空或结果不准确时，提供带项目上下文、命名规范、安全边界和验收命令的 AI 文档重构提示词。

## 验收标准

1. 对含有显式命名 shell 代码块的项目，扫描结果包含 Runme 任务名、相对文件路径、代码行号和命令文本；匿名代码块不进入任务列表。
2. Runme 不在 PATH 时，页面明确提示安装要求，执行按钮不可用。
3. 修改或删除任务文件后，执行会被后端拒绝并提示重新扫描。
4. 任务文件位于项目目录外、扩展名不是 Markdown 或任务名为空时，后端拒绝生成命令。
5. 点击“保存为工作流”后，工作流出现在现有工作流面板中，且可以继续绑定到键位。
6. cargo test projecttasks、图标测试和前端生产构建通过。
7. 扫描为空时自动展开 AI 重构提示词；有结果时仍可手动查看和复制，提示词不得要求 AI 执行发布、部署、迁移或删除操作。
8. 新项目成功扫描后加入历史顶部，已有项目重新扫描时原位更新；旧版单项目路径自动迁移，移除历史不得删除或修改项目文件。
9. Markdown 扫描和 Runme CLI 调用不得阻塞 UI 线程；快速切换多个项目时，过期扫描结果不得覆盖最后选择的项目。

## 非目标

- 不在 DevLauncher 内嵌 Runme 执行器。
- 不自动安装 Runme，不修改项目文件。
- 不在首版统一解析 package scripts、Makefile、mise、go-task、Nx 或 Turbo 等非 Runme 任务格式。
- 不把密码、token、环境变量值或完整 shell 历史保存到 DevLauncher 配置。
