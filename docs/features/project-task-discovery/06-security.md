# 安全设计

## 威胁模型

项目任务本质上是任意 shell 命令。发现任务不等于信任任务；用户打开陌生项目时，必须把执行视为潜在高风险操作。

## 控制措施

- 扫描默认只读，不执行代码块。
- 执行前 Rust 重新校验 root、文件扩展名、canonical path 和任务名称。
- 任务文件必须位于项目目录内，防止通过 ../ 生成外部文件命令。
- 项目名、文件名、任务名在生成 shell 命令时统一进行单引号 quoting。
- 高风险关键词任务在 UI 中显示红色标记并二次确认。
- Runme 缺失时不静默回退到其他执行器。
- 不保存密码、token、环境变量值或任务输出到任务索引。
- 工作流描述只保存项目名、相对文件和行号；当前命令快照属于用户明确选择的本地配置。

## 已知风险

- shell quoting 只能防止 DevLauncher 拼接参数引入的注入，不能改变任务本身的命令语义。
- 风险分类是启发式提醒，可能漏报或误报；用户仍需审阅命令。
- 外部 Runme CLI 和项目脚本具有用户本机权限，建议只执行可信项目中的任务。
- 工作流命令快照在项目迁移、目录改名或命令变更后可能失效，执行前仍会做任务存在性校验。
