# 决策记录

## 2026-07-19：首版选择 Runme 作为外部任务发现器

选择 Runme 的理由：任务以 Markdown 代码块表达，适合从项目文档发现；官方 CLI 提供 list / run 语义；Apache 2.0；可以直接复用 DevLauncher 终端，而不需要把执行器复制进应用。

没有在首版同时接入 Makefile、package scripts、mise、go-task、Nx 或 Turbo，因为它们的任务模型和参数语义不同。先验证“项目任务发现 → 复核 → 终端执行 → 工作流复用”的完整闭环，再扩展多格式适配器。

## 2026-07-19：使用现有 terminal_run，而不是新增后台进程管理

DevLauncher 已有 PTY、输入、输出和窗口生命周期。复用它可以减少重复的进程状态模型，并让 Runme 任务与普通脚本保持一致。代价是首版保存的工作流使用 shell 命令快照，暂未实现独立 Runme 进程的结构化取消和状态回传。

## 2026-07-19：首版任务发现只接受显式 name

显式命名可以避免把文档中的每个代码示例误认为可执行任务，也避免默认执行未审阅的匿名代码块。后续可以增加用户主动开启的“包含匿名任务”选项。

## 2026-07-20：优先使用 Runme CLI 发现未命名代码块

实际 Runme CLI 支持 `list --json --allow-unnamed`，并会为未命名代码块生成可执行名称。DevLauncher 改为优先复用该列表，避免用户已安装 Runme 但项目任务因没有显式 `name` 而显示为空；Runme 不可用时仍保留显式命名代码块的本地解析回退。

## 2026-07-20：恢复显式 name，并提供 AI 重构提示词

真实项目验证发现 `--allow-unnamed` 会把实现计划、目录树、API 示例和输出片段全部转成任务，DevLauncher 项目一次产生 914 条结果。任务发现恢复 Runme 默认的显式 name 规则，不再展示匿名代码块。

为避免旧项目因缺少标记而显示为空，页面提供上下文化的 AI 重构提示词，引导编码助手从项目现有脚本和配置核对真实命令，集中整理 TASKS.md 或 README。提示词只负责文档治理，不授权发布、部署、迁移、删除或 Git 提交。

## 2026-07-22：AI 重构后持久记住 Runme 维护契约

仅整理一次 `TASKS.md` 不能保证后续新增的执行脚本继续可被发现。AI 提示词现在要求将 Runme 任务维护规则写入根目录 `AGENTS.md`：后续新增或修改可重复执行命令时，必须在同一次改动中同步维护显式命名的 Runme 任务。已有 `AGENTS.md` 仅增量合并，不覆盖原有项目规则。

## 2026-07-19：项目目录使用 localStorage

首版不扩展配置 schema，只保存最近一次本地目录偏好；发现结果按次扫描，不产生陈旧索引。待项目 profile、跨机器迁移和项目级权限模型明确后，再进入配置持久化。

## 2026-07-20：本地保存最近扫描项目

单个最近目录无法支持多项目切换，因此扩展为最多 24 条本地项目索引，并兼容迁移旧键。历史只记录路径和扫描摘要，不记录命令、不进入同步配置；点击时始终重新扫描，避免使用陈旧任务结果。移除只清理索引，不操作项目文件。

历史顺序遵循空间稳定原则：新项目首次扫描时加入顶部，已有项目点击或重新扫描只在原位置更新摘要，不按扫描时间重新排序。这样用户可以依靠固定位置切换项目，不会因为一次点击导致列表跳动。

## 2026-07-20：项目切换使用会话缓存，扫描移出 UI 线程

历史项目点击原本会立即遍历 Markdown 并启动 Runme，同时用全局 busy 禁用窗口，造成明显卡顿。现在完整发现结果仅保存在当前窗口内存中，已访问项目点击后立即切换；刷新按钮负责显式重新扫描。Rust 使用 spawn_blocking 执行文件和子进程工作，前端通过请求序号防止慢请求覆盖用户最后选择。
