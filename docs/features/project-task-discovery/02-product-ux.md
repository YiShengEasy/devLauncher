# 产品与交互设计

## 页面结构

~~~text
┌─────────────────────────────────────────────────────────────────┐
│ 项目任务 · Runme Markdown 任务发现器                 窗口控制    │
├──────────────┬──────────────────┬───────────────────────────────┤
│ 已扫描项目    │ 选择项目 / 重扫   │ 任务名称       复制 保存 执行    │
│ 项目 A       │ 项目目录输入       │ 文件、行号、语言                │
│ 项目 B       │ 任务类型筛选       │ 项目 / Runme / 扫描状态          │
│ 项目 C       │ 测试 · 测试项目    │ 命令预览                        │
│              │ 构建 · 构建项目    │ 运行说明 / 扫描提示              │
└──────────────┴──────────────────┴───────────────────────────────┘
~~~

## 核心流程

~~~mermaid
flowchart LR
  A[打开项目任务] --> B[选择项目目录]
  B --> C[扫描 Markdown]
  C --> D{发现命名代码块?}
  D -- 否 --> E[显示提示与安装说明]
  D -- 是 --> F[选择任务]
  F --> G[预览命令与风险]
  G --> H[发送到终端]
  G --> I[保存为工作流]
  I --> J[工作流面板中绑定键位]
~~~

## 状态与反馈

- 初始：选择一个项目目录开始扫描。
- 扫描中：按钮显示扫描中，避免重复请求。
- 成功：显示任务数、扫描文件数、Runme 版本。
- 分类：只显示当前项目实际存在的类型；全部视图按类型分组，类型标签显示任务数量。
- 项目历史：新项目加入顶部，已有项目原位更新；点击访问过的历史项目使用会话缓存立即切换，不跳位，手动刷新才重新扫描。
- 侧栏交互：项目使用 macOS 平面选中行；移除按钮保持固定尺寸和居中图标，仅在悬停时强化危险色。
- 无任务：说明只识别带 name 属性的代码块，并给出示例方向。
- Runme 缺失：任务仍可查看和保存，但执行任务禁用。
- 高风险任务：执行前弹出确认，提示任务名和风险等级。
- 任务失效：后端返回“任务名称已不存在，建议重新扫描项目”。
- 保存成功：提示“已保存为工作流，可在工作流面板中绑定到键位”。
- 后台扫描：只在对应项目行和刷新按钮显示扫描状态，其他项目仍可点击；较早请求完成后不得抢回当前选择。

## 交互原则

- 先展示命令，再执行；执行入口不隐藏实际 shell 命令。
- 任务发现与任务执行分离；扫描是只读动作，执行是明确动作。
- 使用现有终端作为输出承载，避免引入第二套日志和取消模型。
- 保存后回到现有工作流能力，而不是创造平行的“任务快捷方式”模型。
