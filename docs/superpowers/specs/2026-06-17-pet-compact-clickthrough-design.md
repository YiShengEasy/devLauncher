# DevLauncher 电子宠物紧凑窗口与点击穿透设计

日期：2026-06-17

## 设计结论

电子宠物采用“更小默认窗口 + 四角贴片菜单 + 透明区域尽量点击穿透”的方案。

当前宠物窗口原生尺寸是 `284 x 284`，但可见宠物主体只有一百多像素宽，透明区域会挡住桌面点击。新设计把默认窗口缩小到 `152 x 136`，宠物本体也略微缩小。展开菜单时不再显示顶部横排 5 个按钮，而是保留 4 个核心按钮，贴近宠物四角并允许少量覆盖宠物边角。展开窗口约 `172 x 152`，比当前默认窗口小很多。

透明区域点击穿透作为第二层优化：能在 macOS/Tauri 上稳定实现时，让宠物和菜单以外的透明区域不拦截鼠标；如果原生穿透能力只能做到整窗穿透或表现不稳定，则优先保证宠物点击、拖拽和菜单操作稳定。

## 当前前提

当前电子宠物实现集中在：

- `app/src/entry/PetEntryApp.tsx`
- `app/src/index.css`
- `app/src-tauri/tauri.conf.json`
- `app/src-tauri/src/entries.rs`

现状：

- `tauri.conf.json` 中 `pet` 窗口尺寸为 `284 x 284`。
- `PetEntryApp.tsx` 中关闭和展开尺寸都为 `284 x 284`。
- 宠物按钮为 `148 x 132`。
- 宠物普通帧图片宽度约 `168px`，跳跃帧图片宽度约 `188px`。
- 菜单是顶部横向浮层，包含搜索、报告、剪贴板、键盘、动作 5 个按钮。
- 窗口是透明、无边框、置顶、跨工作区显示；透明区域并不自动鼠标穿透。
- `entries.rs` 已有 macOS 私有 API 处理窗口跨空间显示，可在同一处扩展宠物窗口相关原生能力。

## 目标

本次目标：

- 默认状态显著减少电子宠物窗口对桌面点击的遮挡。
- 宠物本体略微缩小，但仍清晰可见、容易点击和拖拽。
- 展开菜单只保留 4 个核心动作：搜索、报告、剪贴板、键盘。
- 4 个菜单按钮贴近宠物四角，允许覆盖宠物边角。
- 去掉“动作/添加”入口。
- 宠物只播放关键帧动画，不再做窗口内跳动位移。
- 尽量让透明区域鼠标点击穿透到底层应用。

## 非目标

本次不做：

- 自定义动作图片上传入口。
- 逐像素读取 PNG alpha 来判断点击命中。
- 跨平台完整非矩形窗口命中系统。
- 改造键盘主窗口布局。
- 新增宠物尺寸设置页。
- 云同步或用户偏好同步。

## 尺寸设计

### 默认状态

默认状态尺寸：

| 项目 | 尺寸 |
| --- | --- |
| 窗口 | `152 x 136` |
| 宠物按钮 | 约 `116 x 102` |
| 普通宠物图片宽度 | 约 `132px` |
| 跳跃帧图片宽度 | 约 `148px` |

默认窗口面积从当前 `284 x 284 = 80656 px²` 降到 `152 x 136 = 20672 px²`，约减少 74%。

### 展开状态

展开状态尺寸：

| 项目 | 尺寸 |
| --- | --- |
| 展开窗口 | 约 `172 x 152` |
| 菜单按钮 | 约 `34 x 30` |
| 菜单按钮数量 | 4 |

展开窗口面积约 `172 x 152 = 26144 px²`，仍比当前默认窗口小约 68%。

## 菜单布局

菜单从顶部横排改为四角贴片布局。

按钮保留：

- 左上：搜索
- 右上：报告
- 左下：剪贴板
- 右下：键盘

删除：

- 动作/添加入口
- 相关自定义动作提示
- 相关本地存储预留逻辑

按钮位置相对宠物主体摆放，允许覆盖宠物图片边角，但避免遮挡宠物中心主体。按钮仍应保留像素风边框、hover 反馈和可点击状态。

## 动画策略

宠物不再做窗口内跳动位移。

保留：

- 普通待机关键帧播放。
- 键盘相关关键帧播放。
- 菜单按钮淡入、缩放、错位出现。
- 从键盘模式切回宠物时的轻量淡入。

调整：

- `keyboardJump` 可以继续作为帧动画名称，但不应让宠物本体在窗口中上下跳动或改变窗口命中区域。
- 菜单展开时宠物位置保持稳定。
- 菜单关闭后窗口恢复默认尺寸，不产生明显位置跳动。

## 点击穿透策略

采用保守点击穿透。

优先级：

1. 宠物点击、拖拽必须可靠。
2. 菜单按钮点击必须可靠。
3. 透明区域尽量不挡桌面点击。

实现方向：

- Web 层：`pet-shell` 默认不扩展交互命中，只有宠物按钮和菜单按钮启用 pointer events。
- 窗口层：在 macOS 上评估是否可以让宠物窗口的透明区域忽略鼠标事件，或者切换窗口忽略鼠标事件状态。
- 如果只能整窗忽略鼠标事件，则不能默认开启整窗穿透，因为宠物会无法点击。
- 如果原生精细命中不可行，保留小窗口和 Web 层 pointer-events 优化作为稳定回退。

## 位置与窗口尺寸切换

默认窗口和展开窗口尺寸不同，需要避免展开/收起时宠物视觉位置跳动。

原则：

- 默认状态存储的位置代表宠物窗口位置。
- 展开菜单时围绕当前宠物中心扩展窗口。
- 收起菜单时再围绕宠物中心缩回默认窗口。
- 位置保存仍沿用当前 `windowPosition.ts` 的 `pet` 位置记录。

实现时可以把尺寸和偏移抽成常量，避免分散在 `PetEntryApp.tsx` 中：

```text
PET_CLOSED_WINDOW_SIZE
PET_OPEN_WINDOW_SIZE
PET_BUTTON_SIZE
PET_IMAGE_WIDTH
PET_KEYBOARD_IMAGE_WIDTH
PET_MENU_BUTTON_SIZE
```

## 验证计划

自动验证：

- 前端测试覆盖宠物窗口尺寸常量。
- 前端测试覆盖菜单项数量和动作集合，不再包含 `custom-action`。
- 前端测试覆盖展开尺寸小于当前 `284 x 284`。
- `npm test`。
- `npm run build`。

Rust 验证：

- 如果新增 macOS 原生窗口穿透命令，为命令状态和参数加最小测试。
- 当前本机可能仍会受 `rustc 1.87.0` 与依赖要求 `1.88+` 限制，Rust 测试结果需要明确记录。

手动验证：

1. 默认宠物明显变小。
2. 默认状态宠物外侧透明区域不再明显挡住桌面点击。
3. 点击宠物能展开菜单。
4. 拖拽宠物仍可移动并保存位置。
5. 展开菜单后 4 个按钮分布在宠物四角。
6. 4 个按钮能分别打开搜索、报告、剪贴板、键盘。
7. 菜单展开和收起时宠物位置不明显跳动。
8. 键盘模式切换后，宠物只播放关键帧，不做窗口内跳动位移。

## 实施边界

本设计先进入实现计划，不直接编码。实现阶段应先完成尺寸和菜单布局调整，再评估点击穿透。若点击穿透风险高，不应牺牲宠物点击和拖拽稳定性，应以小窗口方案作为可靠交付基线。
