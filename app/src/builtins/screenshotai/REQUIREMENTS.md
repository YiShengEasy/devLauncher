# 截图问题报告功能需求

展示名称统一为“截图问题报告”。内部插件 id 仍保留 `screenshotai`，用于兼容现有窗口、路由和存储。

## 目标

把截图变成 AI 能准确理解的视觉问题报告。核心不是做一个完整的通用截图软件，而是在截图后快速表达：

- 操作顺序是什么。
- 哪个区域出现问题。
- 实际效果和期望效果分别是什么。
- 哪些信息需要 AI 重点关注。
- 哪些敏感内容需要隐藏。

## 目标用户

- vibe coding 开发者：截图后直接发给 Codex、ChatGPT、Claude 或同事，请对方定位问题或实现效果。
- 前端/产品开发者：需要把 UI 问题、期望交互、视觉差异讲清楚。
- 测试和支持人员：需要快速提交可复现、可执行的问题说明。

## 核心工作流

1. 使用快捷键截图。
2. 截图完成后进入轻量编辑器。
3. 连续点击截图区域，自动放置 1、2、3 等编号标记。
4. 给每个编号输入一句短备注。
5. 使用箭头、矩形、高亮、打码补充上下文。
6. 一键生成图片和结构化 AI Prompt。
7. 复制到剪贴板或发送到目标 AI 工具。

## MVP 功能

- 区域截图、窗口截图、上一截图区域。
- 自动递增编号标注。
- 编号备注列表，支持改文案、改顺序、删除。
- 箭头、矩形框、高亮、模糊/打码。
- 三段式问题模板：上下文、截图标注、期望结果。
- 一键复制 Markdown Prompt。
- 一键复制带标注截图。
- 保存最近截图和最近 Prompt。

## AI 理解增强

- OCR 识别截图中的文字，并附加到 Prompt。
- 自动提取应用名、窗口标题、URL、时间、屏幕尺寸。
- 支持多截图串联：当前状态、错误状态、期望参考。
- 支持红色问题标记、绿色期望标记、黄色关注标记。
- 支持敏感信息智能打码：邮箱、手机号、token、密钥、URL 参数。

## 输出格式

### 通用 AI Prompt

```md
请根据截图分析这个 UI 问题。

上下文：
- 应用：
- 页面：
- 当前操作：

截图标注：
1.
2.
3.

期望效果：

请帮我判断：
- 可能的问题原因
- 应该检查哪些组件、状态或 API
- 推荐修复方案
```

### Codex Prompt

```md
请根据截图中的编号定位前端问题。优先检查与 1、2、3 标注相关的页面、组件状态和点击事件。目标效果见 4。请先搜索相关代码，再给出修复方案并实现。
```

## 非目标

- 第一版不替代 Snagit、ShareX、CleanShot X。
- 第一版不做复杂图片编辑、滤镜、贴纸和长视频剪辑。
- 第一版不依赖云端上传能力。
- 第一版不默认把截图发送到第三方服务。

## 成功标准

- 用户能在 30 秒内完成截图、编号、说明和复制 Prompt。
- AI 能根据截图编号和说明准确复述问题与期望。
- 同一张截图不需要用户再写长段解释。
- 敏感信息不会被误复制到 Prompt 或标注图里。
