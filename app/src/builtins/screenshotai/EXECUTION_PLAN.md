# 截图问题报告执行计划

展示名称统一为“截图问题报告”。内部插件 id 仍保留 `screenshotai`，用于兼容现有窗口、路由和存储。

## 阶段 0：内置插件落位

- 新增 `app/src/builtins/screenshotai/`。
- 添加 `manifest.ts`、`App.tsx`、需求文档和执行计划。
- 接入内置插件 registry。
- 接入 Tauri window 配置和 `toggle_screenshotai_window` 命令。
- 在按键绑定弹窗中可选择该内置插件。

## 阶段 1：Prompt 工作台

- 提供问题上下文表单。
- 提供编号标注列表编辑。
- 提供期望效果字段。
- 提供一键生成和复制 Markdown Prompt。
- 支持保存最近一次草稿到 `localStorage`。

## 阶段 2：截图导入和手动标注

- 支持从剪贴板读取图片。
- 支持拖拽或选择本地图片。
- 在画布上放置自动递增编号点。
- 支持拖动编号点、删除编号点、编辑备注。
- 支持导出带标注图片。

## 阶段 3：截图捕获

- Windows 首先支持区域截图和窗口截图。
- 支持快捷键启动截图模式。
- 支持复用上一截图区域。
- 截图完成后直接进入标注工作台。

## 阶段 4：AI 理解增强

- OCR 提取截图文字。
- 自动提取窗口标题、应用名、URL、时间、屏幕信息。
- 支持红色问题、绿色期望、黄色关注三种标注类型。
- 支持敏感信息规则识别和一键打码。

## 阶段 5：开发者工作流集成

- 输出 GitHub Issue、Linear/Jira、Codex Prompt 模板。
- 可选附加 repo、branch、commit、当前路径。
- 支持多截图串联说明。
- 支持把 Prompt 直接发送到内置终端或外部 AI 工具。

## 推荐实现顺序

1. 先完成 Prompt 工作台，验证信息结构是否真的能减少沟通成本。
2. 再做图片导入和画布标注，避免过早投入系统截图 API。
3. 最后接 Windows 截图捕获和 OCR，形成完整闭环。

## 技术备注

- 前端优先使用 React canvas 或 SVG overlay 实现标注层。
- 标注数据应独立保存为结构化 JSON，不直接写死到位图。
- 导出时再把原图和标注层合成 PNG。
- 截图和 OCR 属于平台能力，适合放到 Tauri Rust 命令中。
- 敏感信息识别应默认本地执行。
